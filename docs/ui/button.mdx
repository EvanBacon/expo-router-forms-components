A versatile button component with multiple variants and sizes. Uses native Pressable on mobile and semantic HTML button with Radix Slot on web.

## Installation

## Usage

```tsx
import { Button } from "@/components/ui/button";
```

### useButtonContext

A hook to access button state from compound components.

```tsx
import { useButtonContext } from "@/components/ui/button";

function CustomContent() {
  const { variant, size, disabled } = useButtonContext();
  return <Text>Current variant: {variant}</Text>;
}
```

**Returns:**

- `variant: ButtonVariant` - The current button variant
- `size: ButtonSize` - The current button size
- `disabled: boolean` - Whether the button is disabled

## Platform Differences

### Native (iOS/Android)

- Uses React Native Pressable for touch handling
- `active:opacity-80` provides press feedback
- No hover states (touch-based interaction)
- SF Icons or Expo Vector Icons for icons

### Web

- Uses semantic HTML `<button>` element
- Radix Slot support for `asChild` pattern
- Full hover, focus, and active states
- `focus-visible` ring for keyboard navigation
- CSS transitions for smooth state changes
- Lucide React for icons (or any icon library)

## Accessibility

- Proper button semantics on web
- Disabled state removes interactivity
- Focus ring visible for keyboard users
- Supports reduced motion preferences via CSS

## Usage with Links

On web, use `asChild` to render the button as a link:

```tsx
import Link from "next/link";

<Button asChild>
  <Link href="/dashboard">Go to Dashboard</Link>
</Button>;
```

On native with Expo Router:

```tsx
import { Link } from "expo-router";

<Link href="/dashboard" asChild>
  <Button>
    <Button.Text>Go to Dashboard</Button.Text>
  </Button>
</Link>;
```
